# 10. Next Steps

## After Phase 1

Once the basic Town → Hermes flow works, the next implementation steps should be:

## Phase 2: Persistence and memory
Add:
- persistent task storage
- expert memory digests
- knowledge storage
- baseline knowledge loading

## Phase 3: Real-world broadcasts
Add:
- source fetchers
- filtering and scoring
- broadcast event generation
- expert-facing broadcast delivery

## Phase 4: Learning loop
Add:
- learning triggers
- extraction and summarization
- KB update pipeline
- skill update indicators

## Phase 5: Feishu and Web
Add:
- Feishu bot integration
- dynamic task cards
- Web dashboard
- expert hall and task board

## Phase 6: Multi-expert collaboration
Add:
- expert delegation
- collaboration summaries
- structured multi-expert workflows
- better narrative presentation

## Recommended implementation order

1. stabilize Phase 1 skeleton
2. add persistence
3. add broadcasts
4. add learning loop
5. add Feishu / Web
6. expand expert set
7. strengthen world simulation and narrative depth

## Final reminder

Do not expand the town simulation faster than the Hermes-backed execution foundation.

The product works only if:
- Town remains coherent
- Hermes remains central
- learning stays structured
- user experience remains simple
