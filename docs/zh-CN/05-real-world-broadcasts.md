# 05. 现实世界广播

## 为什么需要广播

如果专家系统只在用户提问时被动响应，它会显得静态且缺乏生命感。

所以 Town 应该拥有一个“现实世界广播”机制，把外部世界的变化转化为城镇中的公共信息流。

## 广播来源

广播可以来自：
- GitHub 趋势与 release
- 技术新闻
- 金融市场数据
- 关注公司的公告
- RSS / API / 自定义情报源

## 广播流程

建议流程：

```text
外部数据源
→ 采集
→ 过滤
→ relevance scoring
→ 生成 broadcast event
→ 推送给 Town / 指定专家
```

## 广播的用途

### Town 层用途
- 更新城镇公共信息状态
- 生成“今天镇上发生了什么”
- 影响任务优先级

### 专家层用途
- 更新关注领域动态
- 作为学习触发器
- 形成后续建议或主动提醒

## 例子

比如：
- 某开源框架发布重大版本
- 某公司发布财报
- 某安全漏洞爆发

Town 可以把这些转化为：
- 广播到公共公告栏
- 发送给相关专家
- 触发特定学习任务

## 设计注意点

- 不要把所有外部数据都变成广播
- 需要做 relevance filtering
- 广播应该是“事件”，不是原始信息堆积
- 广播要服务于任务和世界感，而不是制造噪音
