# 06. 知识学习闭环

## 为什么需要学习闭环

如果系统只能执行任务，却不能沉淀有效经验，那么它每次都要从零开始。

因此需要一个学习闭环，把：
- 用户任务
- 外部广播
- 成功案例
- 失败案例
- 高价值结论

逐步转化为可复用知识。

## 学习闭环的核心流程

```text
任务执行 / 外部事件
→ 提取候选知识
→ 价值评估
→ 去重与归类
→ 写入知识库
→ 更新专家摘要或共享知识
```

## 知识类型

### 1. Baseline Knowledge
稳定的基础知识，例如：
- 专家设定
- 方法论
- 常识性规则
- 长期稳定的领域结构

### 2. Personal Knowledge
某个专家自己的经验沉淀，例如：
- 常用判断模式
- 偏好分析框架
- 历史任务总结

### 3. Shared Knowledge
多个专家共享的公共知识，例如：
- 项目背景
- 团队规范
- 常见问题模式
- 可复用解决方案

## 何时触发学习

学习不应该对每次任务都触发。

建议只在以下情况触发：
- 发现高价值新结论
- 问题被完整解决
- 出现重复性任务模式
- 广播事件与长期目标高度相关

## 设计原则

- 学习动作应由 Town 决定是否触发
- 真正的学习执行由 Hermes 完成
- 知识入库前要有质量过滤
- 知识应该可追踪、可回滚、可分类

## 最终目标

系统随着使用会越来越像一个有经验积累的专家社区，而不是重复回答的短期会话机器人。
