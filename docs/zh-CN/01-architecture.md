# 01. 架构设计

## 分层架构

系统采用两层核心结构：

```text
用户 / CLI / 飞书 / Web
        ↓
   Town Orchestration Layer
        ↓
   Hermes Execution Layer
        ↓
模型 / 工具 / 技能 / 外部数据 / 知识库
```

## Town 层职责

Town 是用户体验与专家组织层，负责：
- 用户意图理解
- 专家选择与路由
- 城镇世界观和公共广播
- 任务看板
- 结果的人格化表达
- 记忆摘要

## Hermes 层职责

Hermes 是执行内核，负责：
- 任务执行
- 工具调用
- 技能调用
- 模型选择
- 安全策略
- 外部信息接入
- 学习动作执行

## 边界规则

### Town 可以做的事
- 选择哪个专家来处理任务
- 组装任务上下文
- 决定是否触发学习
- 以专家身份对结果进行包装

### Town 不该做的事
- 自己实现完整执行引擎
- 直接接管工具调度
- 把 Hermes 替换成一套新的 runtime

### Hermes 必须负责的事
- 真正完成任务
- 真正管理工具与技能
- 处理执行安全
- 产出结构化结果

## 推荐心智模型

你可以把系统理解成：

- **Town = 操作系统的桌面环境 / 世界层**
- **Hermes = 真正执行程序的内核**

Town 决定“谁来做、怎么呈现”；Hermes 决定“具体怎么完成”。
