# 16. 部署与运维

## 目标

当 Town × Hermes 从概念验证进入持续运行阶段，就必须考虑部署与运维问题。

## 部署层次

建议至少区分三层：

### 1. Development
用于本地开发与快速测试。

### 2. Staging
用于功能联调、集成验证与风险测试。

### 3. Production
用于真实用户与长期运行。

## 核心运维关注点

### 1. 日志
需要收集：
- 请求日志
- 任务日志
- 执行日志
- 错误日志
- 学习写入日志

### 2. 指标
至少监控：
- 请求量
- 成功率
- 平均延迟
- 工具调用次数
- 成本估算
- 失败任务数

### 3. 告警
建议在以下场景告警：
- 执行失败率突然升高
- 某工具连续异常
- 预算异常消耗
- 学习写入异常增长

### 4. 配置管理
配置项应独立管理，例如：
- 模型配置
- 工具开关
- 权限矩阵
- 预算阈值
- 外部 API Key

## 发布策略

建议采用逐步发布：
- 本地验证
- Staging 验证
- 小流量发布
- 全量发布

## 故障处理

系统应支持：
- 快速定位失败任务
- 查看执行链路
- 暂停高风险功能
- 回滚知识写入
- 人工介入恢复

## 原则

运维不是系统完成后的附加项，而是长期可运行能力的一部分。
