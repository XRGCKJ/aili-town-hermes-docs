# 09. 记忆与摘要

## 为什么不保存完整一切

如果系统把所有对话和执行细节都原样长期保存：
- 成本会快速上升
- 检索质量会下降
- 专家人格会被噪音污染

因此更合理的做法是：

> **保存高价值摘要，而不是无差别保存全部原始上下文。**

## 记忆分层

### 1. Session Memory
当前会话上下文，短期使用。

### 2. Expert Summary Memory
每个专家的摘要记忆，例如：
- 最近解决了哪些问题
- 当前关注哪些主题
- 有哪些稳定偏好

### 3. Shared Town Memory
整个城镇共享的公共记忆，例如：
- 项目背景
- 用户长期目标
- 团队规则
- 关键事件

## 摘要的用途

摘要不是历史记录备份，而是帮助系统更好地：
- 路由
- 对齐风格
- 避免重复劳动
- 维持连续感

## 什么值得进入记忆

适合进入记忆的内容包括：
- 长期稳定偏好
- 已验证的重要结论
- 重复出现的目标
- 高价值项目背景
- 专家之间的稳定分工

## 什么不该进入记忆

不应该长期保存：
- 瞬时噪音
- 低价值闲聊
- 未验证猜测
- 大量原始日志

## 原则

记忆系统应该追求：
- 少而精
- 易更新
- 易检索
- 易回滚
