# 04. Town 编排层

## 角色定位

Town 是用户能感知到的系统表层。

它负责把冷冰冰的执行过程，组织成一个：
- 有人物感
- 有协作感
- 有世界状态
- 有任务流转

的专家城镇体验。

## Town 的核心职责

### 1. 意图识别
识别用户是在：
- 提问
- 请求调试
- 请求研究
- 请求写作
- 请求投资判断
- 请求持续跟踪

### 2. 专家路由
根据任务类型、难度和上下文，决定交给哪个专家。

### 3. 上下文组装
在送入 Hermes 之前，Town 需要组装：
- 用户需求
- 专家 persona
- 记忆摘要
- 相关广播
- 可选知识上下文

### 4. 任务看板
Town 管理任务状态，例如：
- new
- routed
- executing
- waiting
- done
- archived

### 5. 结果叙事包装
Hermes 返回的是执行结果；Town 负责把它转成适合用户阅读的专家回应。

## Town 的设计边界

Town 不应：
- 直接替代 Hermes 执行工具链
- 直接承载所有模型策略
- 变成一个新的 agent runtime

Town 的强项是：
- 编排
- 表达
- 世界组织
- 专家身份管理

## Town 的价值

Town 让系统从“工具型 agent”升级为“可感知、可协作、可持续运转的专家世界”。
