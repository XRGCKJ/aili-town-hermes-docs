# 07. 用户交互层

## 目标

系统应该支持多个入口，而不要求用户理解内部架构。

支持的入口包括：
- CLI
- 飞书
- Web

## 核心原则

用户表达目标，而不是理解路由逻辑。

系统应自动处理：
- 任务分类
- 专家选择
- 学习触发
- 结果包装

## CLI

CLI 是最适合早期验证架构的入口。

建议命令：
- `town ask "为什么这个 Python 项目启动失败？"`
- `town ask --expert intel "总结这个 README"`
- `town experts`
- `town task list`

## 飞书

飞书适合团队协作与异步使用。

潜在能力：
- `@town` 自动路由
- `@expert` 定向请求
- 任务状态卡片
- 学习通知
- 每日报告

## Web

Web 适合可视化与运维观察。

潜在能力：
- 专家大厅
- 广播流
- 任务看板
- 知识库
- 执行链路视图

## 输出设计

一个对用户有价值的结果通常包含三层：
1. 直接结论
2. 专家视角或协作摘要
3. 可选的来源与细节

这样既保留可用性，也保留系统智能感。
